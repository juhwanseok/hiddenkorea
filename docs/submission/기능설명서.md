# 숨은한국 (HiddenKorea) — 기능설명서

> **2026 관광데이터 활용 공모전 · ②-2 웹·앱 구현 부문**
> 지정과제 **2번** — 유명 관광지 중심의 관광객 쏠림(오버투어리즘) 문제
> 1차 심사 제출용 · 작성일 2026-07-13

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | **숨은한국 (HiddenKorea)** |
| 한 줄 정의 | 붐비는 곳 말고, 숨은 한국 — 방문 예정일의 관광지 혼잡도를 AI로 예측하고 "비슷하지만 한적한" 대안·코스를 추천하는 오버투어리즘 분산 서비스 |
| 지정과제 | 2번: 유명 관광지 쏠림으로 인한 특정 명소의 과밀화(오버투어리즘) |
| 타깃 | 인파를 피해 여유로운 국내 여행을 원하는 자유여행객 |
| 서비스 형태 | 반응형 웹 (모바일 우선) |
| 핵심 가치 | 수요를 **지우지 않고 옮긴다** — 여행자 경험 개선 + 저평가 지역 활성화 동시 달성 |

### 문제 정의
국내 관광 수요가 소수 명소에 집중되어 ① 방문객 경험 저하(대기·인파) ② 지역 주민 피해 ③ 인근 저평가 관광지의 기회 상실이라는 3중 손실이 발생한다. 여행자는 "덜 붐비는 곳에 가고 싶지만" **언제 어디가 붐비는지, 그 대신 어디를 가면 좋은지** 알 방법이 없다.

### 해법
방문 *예정일*의 혼잡도를 **예측**하고, 취향은 유지하되 혼잡만 제거한 **대안 명소**와 **코스·일정**을 근거와 함께 제시한다.

## 2. 핵심 해시태그 ↔ 구현 기능 매핑

지정과제 참고자료의 해시태그(`#혼잡도 분석 #과밀지역 우회 #숨은 명소 #코스 추천`)를 실제 구현 기능과 1:1로 대응한다.

| 해시태그 | 구현 기능 | 관련 API |
|---|---|---|
| **#혼잡도 분석** | 하이브리드 혼잡 예측 엔진 + 30일 혼잡 달력 + 홈 "평소보다 한적한 명소" | `/api/congestion`, `/api/highlights` |
| **#과밀지역 우회** | 붐빔 감지 시 대안 추천 + 자기조절(혼잡한 대안 자동 제외) | `/api/alternatives` |
| **#숨은 명소** | 분류체계+임베딩 기반 유사 대안 매칭 + 자기평균 대비 한적 명소 발굴 | `/api/alternatives`, `/api/highlights` |
| **#코스 추천** | 동선 최적화 코스 생성 + 지역·기간·장르 기반 가이드북형 다일정 | `/api/course`, `/api/itinerary` |

## 3. 기능별 상세

### 기능 1. 하이브리드 혼잡 예측 엔진 `#혼잡도 분석`
**무엇을** — 관광지·날짜를 입력하면 그날의 혼잡 지수(0~100)·5등급(여유~매우혼잡)과 향후 30일 혼잡 달력을 제공한다.

**어떻게 (차별화 핵심)** — 공사 데이터를 AI로 전국 확장한 2단 구조:
- **커버 관광지(7,445곳)**: 한국관광공사 "관광지 집중률 방문자 추이 예측" API의 예측값을 직접 사용 → 출처 배지 `공사 예측`
- **미커버 관광지**: 자체 **LightGBM 회귀 모델**이 (지역·카테고리·요일·달력) 피처로 혼잡을 근사 → 출처 배지 `숨은한국 AI 추정`. 커버리지 밖 지역(예: 광주·세종)까지 30일 예보 제공

**사용자 플로우** — 검색 → 관광지 선택 → 혼잡 지수·등급 + 30일 달력(요일 헤더, 색상 히트맵) → 날짜 클릭 시 즉시 전환

**신뢰성** — 예측 출처를 배지로 투명하게 구분, 색+텍스트 병기(접근성)

---

### 기능 2. 대안지 매칭 — "비슷하지만 한적한 곳" `#과밀지역 우회` `#숨은 명소`
**무엇을** — 원하는 명소가 붐비면, 같은 종류의 매력을 가지되 덜 붐비는 대안 3곳을 추천 근거(유사도·예상 혼잡·거리)와 함께 제시한다.

**어떻게** —
1. **후보 생성**: TourAPI 공식 분류체계(`lclsSystm`) 중분류로 "같은 종류의 명소"를 보장 (예: 경복궁 → 고궁·역사 명소)
2. **랭킹(HiddenScore)** = α·의미 유사도(문장 임베딩) + β·(1−혼잡) + γ·접근성(거리) + δ·정보 품질
3. **자기조절**: 혼잡 임계(70) 초과 후보는 자동 제외 → "붐비는 대안"을 거르는 것이 분산 목적에 부합
4. **노이즈 정제**: 하위 구성물(0.4km 이내)·학교·동상 등 비목적지 제외

**결과 예시** — 경복궁(혼잡 66) → 경희궁·창경궁 등 궁궐(전부 혼잡 45 이하)

**타지역 대안 — "이 지역은 어떠세요?" `지역 간 분산`** — 근처뿐 아니라 **전국에서 비슷한 느낌(공식 소분류 매칭)이면서 다른 지역이라 더 한적한 명소**를 지역 배지와 함께 추천. 지자체별 유사 관광지가 늘어나는 흐름을 활용해 수도권 집중을 지방으로 분산 유도. (예: 거제식물원 → 전북 정읍 허브원·경기 서후리숲). 시도별 최고 1곳씩 뽑아 지역 다양성 보장.

---

### 기능 3. 코스 추천 — 동선 최적화 `#코스 추천`
**무엇을** — 선택한 명소들로 이동 거리를 최소화한 하루 코스를 생성하고 지도·타임라인·카카오맵 길찾기로 제공.

**어떻게** — 거리 행렬(Haversine) 기반 최근접 이웃 + **2-opt 최적화**로 방문 순서 결정, 도착 시각 자동 계산, 카카오맵 SDK로 마커·경로선 표시.

---

### 기능 4. 여행 일정 추천 — 가이드북형 다일정 `#코스 추천`
**무엇을** — 지역(시도·시군구) + 기간(2박3일 등) + 장르(관광·식도락·쇼핑·레포츠·문화, 다중 선택)를 고르면, 실제 여행 가이드처럼 날짜별 일정을 짜준다.

**어떻게** — 하루를 시간대 슬롯으로 구성:
`09:30 오전 관광 → 12:00 점심 → 14:00 오후 관광 → 15:30 카페 → 17:00 관광·쇼핑 → 18:30 저녁`
- 식사는 실제 식사 시간에 배치, 카페/맛집은 **소분류 코드(FD05=카페)**로 정확히 구분
- 각 슬롯은 그날 **덜 붐비는 곳**을 중복 없이 선택 → 분산 목적 반영
- 관광만 골라도 식사 자동 삽입, 식도락만 골라도 관광 자동 연계

---

### 기능 5. 홈 발견 — "오늘 평소보다 한적한 명소" `#혼잡도 분석` `#숨은 명소`
**무엇을** — 첫 화면에서 선택한 날짜 기준으로 **자기 30일 평균보다 한산할 명소**를 지역별로 큐레이션. "평소 −80%" 같은 배지로 지금이 방문 적기임을 보여준다. 날짜·새로고침마다 유동적으로 갱신.

**왜** — 검색 전에도 "이 날 가면 좋은 숨은 타이밍"을 데이터로 제안 → 분산을 첫 화면부터 유도.

## 4. 한국관광공사 OpenAPI 활용 명세 `데이터 활용`

> 공사 OpenAPI가 서비스 **핵심 경로**에 필수적으로 사용됨 — 없으면 서비스가 성립하지 않는다.

| 공사 OpenAPI | 활용 오퍼레이션 | 서비스 내 역할 |
|---|---|---|
| **국문 관광정보 서비스 (TourAPI, KorService2)** | `areaBasedList2` | POI 마스터 **50,674건** 수집(제목·좌표·분류·이미지) — 검색·매칭·코스·일정의 기반 |
| 〃 | `detailCommon2` | 관광지 상세 설명(overview) — "이곳에서는" 소개 |
| 〃 | `ldongCode2` | 시도·시군구 행정표준코드 — 지역 선택·혼잡 데이터 조인 |
| **관광지 집중률 방문자 추이 예측** | `tatsCnctrRatedList` | 전국 **7,445개 관광지 × 30일** 혼잡 예보 — 혼잡 엔진 1차 소스 |

- 외부 보강(선택): 카카오맵 SDK(지도 표시)
- 라이선스: TourAPI 공공누리 준수, 출처 표기(푸터 "데이터: 한국관광공사 TourAPI")

## 5. 기술 구성 (구현성)

```
[Next.js 프론트엔드]  ── REST ──  [FastAPI 백엔드]  ──  [SQLite/Postgres]
   검색·달력·지도                 4대 API + 홈/지역/일정        POI 5만 · 집중률 22만행 · 임베딩
                                      │
             ┌────────────────────────┼─────────────────────────┐
        혼잡 엔진               대안 매칭 엔진              코스·일정 엔진
   (집중률 API + LightGBM)   (분류필터 + 문장임베딩)      (2-opt + 슬롯 배치)
```
- 프론트: Next.js 16 + TypeScript + Tailwind, 카카오맵 SDK
- 백엔드: FastAPI(Python), REST API 8종
- AI: LightGBM(혼잡 근사) · 문장 임베딩(대안 매칭) · (선택) LLM 문장화
- 데이터 파이프라인: 공사 API 수집·적재·모델 학습 배치
