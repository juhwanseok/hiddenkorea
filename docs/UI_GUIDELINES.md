# UI_GUIDELINES — 숨은한국 화면 설계

> STEP 8 산출물. 프론트는 사용자 약점 → **화면 수를 4개로 고정**하고 shadcn/ui 기성 컴포넌트만 사용. 커스텀 CSS 최소화.

## 사용자 플로우 (해피 패스 60초)

```
[홈] 목적지·날짜 입력
  → [혼잡 예보] 지수·등급 + 30일 히트맵 캘린더
  → (붐빔 이상이면) "한적한 대안 보기" CTA
  → [대안 추천] 카드 3장 (유사도·혼잡·거리·이유)
  → 대안 선택 → [코스] 지도 + 타임라인 + 공유 링크
```

## 화면 명세

### S1. 홈 (`/`)
- 히어로: 서비스 한 줄 ("붐비는 곳 말고, 숨은 한국") + 검색창(관광지/지역 자동완성) + 날짜 피커(오늘~+30일)
- 하단: "지금 가장 붐비는 곳 TOP 5 / 가장 한적한 보석 TOP 5" 실데이터 티저 (첫인상에서 데이터 서비스임을 증명)

### S2. 혼잡 예보 (`/place/[contentId]?date=`)
- 상단: POI 이미지·이름·주소 (TourAPI)
- 혼잡 게이지 (0~100 + 5등급 색상: 초록→빨강) + `source` 배지("공사 예측" / "숨은한국 AI 추정" — 투명성이 신뢰 포인트)
- 30일 히트맵 캘린더 (셀 클릭 시 날짜 변경)
- 등급 '붐빔' 이상 → 주황 배너 CTA: "이 날 비슷한데 한적한 곳 3곳 찾기"

### S3. 대안 추천 (`/alternatives`)
- 원본 POI 요약 핀 + 대안 카드 ×3: 이미지 / 이름 / **"유사도 91% · 예상 혼잡 23% · 12km"** / LLM 한 줄 이유
- 카드 액션: [여기로 바꾸기] [코스에 추가]
- 우측(모바일 하단): 카카오맵 미니맵에 원본·대안 동시 표시

### S4. 코스 (`/course/[shareId]`)
- 좌: 타임라인 (09:00 A → 이동 20분 → 11:00 B …) + LLM 서사 한 단락
- 우: 카카오맵 경로 폴리라인 + 각 지점 혼잡 뱃지
- 상단: [링크 복사] [카카오맵에서 열기] — shareId는 로그인 없는 영속 공유

## 디자인 규칙

- 모바일 퍼스트 (심사위원은 폰으로 열 확률 높음), 데스크톱은 2컬럼 확장
- 색: 혼잡 스케일 = `#22c55e → #eab308 → #f97316 → #ef4444 → #991b1b`, 브랜드 = teal 계열 1색만
- 로딩: 모든 API 호출에 skeleton — **스피너 무한 대기 절대 금지** (구동 안정성 30점)
- 에러: 실패 시에도 빈 화면 금지, 캐시된 결과 또는 안내 문구 + 재시도 버튼
- 접근성: 등급을 색+텍스트 병기 (색맹 대응 — 심사 디테일 어필)
