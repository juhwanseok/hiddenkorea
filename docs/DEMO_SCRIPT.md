# DEMO_SCRIPT — 3분 시연 시나리오

> 1차 심사 영상·최종 PT 라이브 공용. 데모 데이터가 아니라 **실서비스 실데이터**로 수행.

## 사전 조건
- 시연 POI: 경복궁 (혼잡 예측이 높게 나오는 주말 날짜 사전 확인)
- 백업: 동일 시나리오 사전 녹화본 + 오프라인 스크린샷 덱 (네트워크 장애 대비)

## 타임라인
| 시간 | 화면 | 대사 골자 |
|---|---|---|
| 0:00~0:20 | 슬라이드 1장 | "작년 한국 방문객의 발길은 상위 몇 개 명소에 몰렸습니다. 여행자는 인파에 지치고, 옆 동네 보석은 비어 있습니다." |
| 0:20~0:50 | S1→S2 | 경복궁 + 10월 첫 토요일 검색 → **"혼잡 87, 매우 혼잡"** 게이지 + 30일 캘린더. "공사 예측 데이터와 저희 ML 모델이 전국 어느 관광지든 이 예보를 만듭니다" |
| 0:50~1:40 | S3 | CTA 클릭 → 대안 3곳. 카드 읽기: "유사도 91%, 예상 혼잡 23%, 12km — 같은 조선 궁궐의 고즈넉함은 그대로입니다." **"왜 이 추천인지 근거를 함께 보여주는 것"** 강조 |
| 1:40~2:20 | S4 | 대안 반영 코스 생성 → 지도 동선 + 타임라인 + 카카오맵 열기 → 공유 링크 복사 |
| 2:20~2:50 | 슬라이드 | 검증: "서울 121개 명소 실측 대비 등급 일치율 N%" 그래프 + 베타 수용률 |
| 2:50~3:00 | 마무리 | "숨은한국은 수요를 지우지 않고 옮깁니다. 여행자도, 지역도 이깁니다." |

## 리허설 체크
- [ ] 시연 경로 전 구간 캐시 워밍 (당일 아침)
- [ ] 폰 화면 미러링 리허설 (심사장 프로젝터 비율 16:9)
- [ ] 실패 분기: API 지연 시 즉시 녹화본 전환 멘트 준비 ("실서비스 영상으로 이어가겠습니다")
