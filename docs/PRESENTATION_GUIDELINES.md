# PRESENTATION_GUIDELINES — 최종 PT (통과 시)

> 배점: 적정성 30 / 완성도 30 / 실용성 25 / 발표 15. 통합심사에서 ①부문 상위 5팀과 대상 경쟁.

## 덱 구조 (10~12장, 발표 7분 가정 — 실제 시간은 OT 공지 따름)
1. 후킹: 쏠림 현상 한 장 (데이터랩 수치 지도) — 문제의 크기
2. 문제 정의: 여행자·지역·저평가 명소의 3중 손실
3. 해법 한 장: "예측 → 대안 → 코스" 3단 그림
4. **라이브 데모** (3분, DEMO_SCRIPT)
5. AI 구조: 하이브리드 혼잡 엔진 다이어그램 (공사 데이터 + 자체 ML)
6. 검증: 정확도 그래프 + 골드셋 nDCG — "측정된 AI"
7. 데이터 활용 명세: 공사 OpenAPI 4종이 핵심 경로에 (다이어그램 강조)
8. 실사용 증거: 베타 지표
9. 발전성: 다국어 → 지자체 B2B 로드맵 + 운영비 구조
10. 마무리: 미션 한 문장

## 발표 수칙
- 기술 용어는 5장·6장에만 격리 — 나머지는 심사위원 전원(정책·관광 배경 포함)이 이해할 언어
- 모든 주장 뒤에 숫자: "정확해요" ✗ → "실측 대비 N% 일치" ○
- Q&A: JUDGING_STRATEGY.md 예상 질문 7종 답변 암기, 모르면 "측정해보지 않았습니다. ~로 검증할 계획입니다"로 정직하게
- 통합심사 특성: ①부문 팀들은 3월부터 개발(기간 2배) → **완성도 총량 경쟁 대신 "AI 깊이 + 검증 문화"로 차별화**하는 포지셔닝
