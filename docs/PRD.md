# PRD — 숨은한국 (HiddenKorea)

> 2026 관광데이터 활용 공모전 ②-2 웹·앱 구현 부문 / **지정과제 2** (오버투어리즘 분산)
> 버전 1.0 (2026-07-08) · 이 문서는 모든 개발 의사결정의 최상위 기준이다.

## 1. 문제 정의

**지정과제 원문**: "유명 관광지 중심의 관광객 쏠림 현상으로 인한 특정 명소의 오버투어리즘(과밀화) 문제"

- 한국 관광은 소수 명소(경복궁, 성산일출봉, 감천문화마을 등)에 수요가 집중되어 ① 방문객 경험 저하(대기·인파) ② 지역 주민 피해 ③ 인근 저평가 관광지의 기회 상실이라는 3중 손실이 발생한다.
- 여행자는 "덜 붐비는 곳에 가고 싶지만" **어디가 언제 붐비는지, 그리고 그 대신 어디를 가면 좋은지**를 알 방법이 없다. 네이버·구글의 혼잡 정보는 실시간 위주·음식점 중심이고, 관광지 단위의 **미래 예측**과 **대안 추천**은 제공하지 않는다.

**우리의 해법**: 방문 *예정일*의 관광지 혼잡도를 예측하고, 취향은 유지하면서 혼잡만 제거한 **"비슷하지만 한적한" 대안지를 AI가 근거와 함께 추천**, 전체 코스를 재구성해주는 웹 서비스.

## 2. 시장 및 시의성

- 오버투어리즘은 문체부·관광공사의 공식 정책 어젠다(지역관광 활성화, 분산 정책). 심사기준 "시의적절성" 직결.
- 국내여행 경험률 90%+, 그러나 방문지 집중도는 상위 10개 시군구에 편중 — 공사 데이터랩 수치로 발표 자료에서 정량 제시할 것.
- 경쟁 서비스: 네이버지도(실시간 혼잡, 예측·대안 없음), 트리플/마이리얼트립(일정 추천, 혼잡 미반영). **"예측 + 대안 매칭 + 코스 재구성"의 결합은 공백 시장**.

## 3. 타깃 사용자 & Pain Point

| 페르소나 | 상황 | Pain Point |
|---|---|---|
| P1. 주말 나들이 커플 (28세, 서울) | 토요일 경복궁·북촌 계획 | "가면 사람에 치여요. 근데 딴 데는 어디가 좋은지 몰라요" |
| P2. 가족 여행객 (40대, 아이 2) | 여름휴가 제주 3박4일 | 아이 동반이라 대기·인파가 여행 파괴 요인 |
| P3. 지자체 관광 담당자 (확장 타깃) | 분산 정책 수립 | 어디로 얼마나 분산되는지 데이터 없음 |

## 4. User Story (MVP 범위 = US1~US5)

- **US1**: 여행자로서, 목적지와 방문 예정일을 입력하면 **그날의 혼잡도 예보(0~100 지수 + 등급)**를 보고 싶다.
- **US2**: 여행자로서, 내 목적지가 붐빈다면 **비슷한 매력의 한적한 대안지 3곳**을 이유와 함께 추천받고 싶다. ("경복궁과 유사도 91% — 같은 조선 궁궐 양식, 예상 혼잡 23%")
- **US3**: 여행자로서, 대안지를 반영한 **하루 코스(동선 최적화)**를 받아 카카오맵으로 바로 이동하고 싶다.
- **US4**: 여행자로서, 지역 전체의 **혼잡도 히트맵 캘린더**(향후 30일)를 보고 여행 날짜 자체를 조정하고 싶다.
- **US5**: 여행자로서, 추천 결과를 **링크로 공유**하고 싶다.
- US6 (post-MVP): 실시간 위치 기반 "지금 근처의 한적한 곳", 다국어(영/일/중), 지자체 대시보드.

## 5. 핵심 기능 ↔ 해시태그 매핑 (1차 심사 기능설명서 대비 — 공고 요구사항)

| 해시태그 (공고 예시) | 구현 기능 | US |
|---|---|---|
| #혼잡도 분석 | 하이브리드 혼잡도 엔진 (공사 집중률 예측 API + 자체 ML 근사 모델) | US1, US4 |
| #과밀지역 우회 | 혼잡 임계치 초과 시 자동 대안 제시 + 코스 재구성 | US2, US3 |
| #숨은 명소 | 임베딩 유사도 기반 대안지 매칭 (HiddenScore 랭킹) | US2 |
| #코스 추천 | LLM 코스 생성 + 거리 기반 동선 최적화 | US3 |

## 6. MVP 정의 (1차 기능심사 제출 기준)

**반드시 동작해야 하는 것 (Definition of Done)**:
1. 지역/관광지 검색 → 날짜별 혼잡도 예보 표시 (전국, TourAPI POI 기준)
2. 대안지 추천 3건 + 추천 근거(유사도·혼잡도·거리) 표시
3. 1일 코스 생성 + 지도 표시 (카카오맵 SDK)
4. 30일 혼잡 캘린더 히트맵
5. 배포된 공개 URL에서 **로그인 없이** 크래시 없이 동작 (심사위원이 직접 조작)

**의도적으로 하지 않는 것**: 회원가입/로그인, 리뷰·커뮤니티, 예약, 네이티브 앱, 실시간 GPS 트래킹. (완성도 30점을 위해 범위를 좁힘)
