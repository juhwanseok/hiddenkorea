# JUDGING_STRATEGY — 심사 대응 전략

> 심사기준 원문(공고문 7p) 기준. 항목별 "점수를 만드는 행동"을 정의한다.

## 1차 심사 (서면+기능, 100점)

### 서비스 구현성 30점 — 최대 배점
- [ ] 심사위원이 URL 접속 → 로그인 없이 60초 내 핵심 가치 도달
- [ ] 크래시·무한로딩·빈 화면 제로 (W7 품질 스프린트 + keep-alive)
- [ ] 과제 연계기능 "완결성": 해시태그 4종(#혼잡도분석 #과밀지역우회 #숨은명소 #코스추천) 각각이 동작하는 기능으로 존재 — 기능설명서에서 1:1로 지목
- 함정 회피: 미완성 기능은 숨긴다 (반쯤 되는 기능 > 없는 기능이 아니라, **없는 기능 > 반쯤 되는 기능**)

### 서비스 기획력 30점
- 타당성: 오버투어리즘 정량 근거 (데이터랩 집중도 수치) 제시
- 구체성: 페르소나·시나리오가 신청서와 서비스 화면에서 일치
- 독창성 문장 (그대로 쓸 것): "혼잡을 피하라고만 하는 서비스는 있지만, **취향을 유지한 채 혼잡만 제거한 대안을 근거와 함께** 주는 서비스는 없습니다"

### 데이터 활용 적절성 20점
- 공사 OpenAPI 3종+ 사용을 명세로 증빙: TourAPI(POI·행사) + 집중률 예측 + 지역별 방문자수 (+연관 관광지)
- **핵심 경로 증명**: "TourAPI 없으면 서비스가 성립하지 않는다"를 아키텍처 다이어그램으로
- 가점 요소: 공사 데이터를 소비만 하지 않고 **AI로 확장**(커버리지 밖 POI 근사)했다는 서사

### 서비스 발전성 20점
- 지속성: 무료 티어 기반 월 2만원 이하 운영비 → "공모전 후에도 유지 가능" 명시
- 확장성: 다국어(방한객) → 지자체 B2B 로드맵 + 데이터 파이프라인이 소스 추가에 열린 구조

## 최종 PT 심사 (통과 시, 100점)
- 적정성 30: 문제(쏠림 데이터) → 해법(3계층 AI) → 검증(citydata 정확도) 논리 사슬
- 완성도 30: **라이브 데모** (사전 녹화는 백업). DEMO_SCRIPT.md
- 실용성 25: 베타 지표 (수용률·SUS) — 실사용자 증거가 학부생 팀 차별화 결정타
- 발표 15: PRESENTATION_GUIDELINES.md

## 예상 질문 Top 7 + 답변 골자
1. "혼잡도 예측이 맞는지 어떻게 아나?" → 서울 121개소 실측 검증 리포트 + 불확실성 UI 표시
2. "네이버 혼잡도와 뭐가 다른가?" → 네이버는 현재·장소단위, 우리는 미래 30일+대안 매칭+코스 재구성
3. "공사 API가 이미 예측을 주는데 당신의 AI는 뭘 했나?" → 커버리지 밖 전국 POI 확장(하이브리드), 매칭·코스는 전부 자체 AI
4. "사람들이 정말 대안으로 가나?" → 베타 수용률 수치 + 분산 인센티브(쿠폰) 로드맵
5. "숨은 명소가 알려지면 거기도 붐비지 않나?" → 혼잡도가 피드백 루프로 반영되는 자기 조절 구조 (추천 자체가 혼잡 데이터 기반이므로)
6. "수익 모델은?" → 단기 무료 + 중기 지자체 B2B 리포트 + 장기 숙박·쿠폰 제휴
7. "데이터가 틀리면 사용자 피해는?" → 예측임을 명시(source 배지), 등급 표현으로 과신 방지
