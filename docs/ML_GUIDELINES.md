# ML_GUIDELINES — 숨은한국 AI 설계

> STEP 6 산출물. AI는 3계층이며, **각 계층은 독립적으로 실패해도 서비스가 성립해야 한다.**

## 계층 1 — 혼잡도 예측 (핵심 ML)

### 무엇을, 왜
- 공사 집중률 예측 API가 커버하지 않는 POI의 혼잡도를 근사하는 **자체 모델**. "공사 데이터를 AI로 전국 확장"이 심사 스토리의 척추.
- **모델: LightGBM 회귀** (선정 이유: 표 형식·소규모 데이터에서 딥러닝 대비 우위, 학습 수분, SHAP/피처중요도로 설명 가능 → 발표자료 소재. 파인튜닝·GPU 불필요)
- 학습이 필요한가? **필요** — 단, 사전학습 모델이 아니라 공개 데이터로 처음부터 학습하는 경량 모델.

### 타깃 설계 (중요)
- 직접적인 POI별 혼잡 정답은 없다 → **프록시 타깃**: (a) 집중률 API 커버 POI에서는 그 예측치를 정답으로 지도학습(지식 증류 관점) (b) 서울 121개 citydata 실측치로 보정·검증.
- 예측 단위: (POI, 날짜) → 혼잡지수 0~100.

### 피처 (DATA_GUIDELINES.md의 수집 파이프라인과 1:1 대응)
| 그룹 | 피처 |
|---|---|
| 시공간 | 시군구 일별 방문자수(데이터랩 API) 최근 4주 평균·전년 동기, 월, 요일, 공휴일/연휴, 방학 |
| 기상 | 단기예보(기온·강수확률·하늘상태) |
| POI | 카테고리(cat1~3), 실내/실외 추정, 이미지·정보 완결도 |
| 이벤트 | 반경 5km 내 행사 개최 여부(TourAPI 행사) |

### Evaluation
- 지표: **MAE, Spearman 순위상관** (절대값보다 "어디가 더 붐비나" 순위가 서비스 본질)
- 프로토콜: 시간 기반 분할 (마지막 4주 홀드아웃). citydata 실측 대비 등급 일치율 리포트.
- 합격선: 등급(5단계) 인접 허용 정확도 ≥ 80% — 미달 시 등급 수를 3단계로 축소 (graceful degradation).

## 계층 2 — 대안지 매칭 (임베딩)

- **임베딩**: TourAPI `overview` 텍스트 → multilingual-e5-small (384dim, 로컬 무료). 전처리: HTML 태그 제거, 500자 절단, "query: "/"passage: " 프리픽스 규약 준수.
- pgvector HNSW 인덱스, cosine. 후보 필터(동일 대분류 + 거리)를 SQL로 먼저 → 벡터 랭킹 (전수 스캔 금지).
- HiddenScore 가중치 튜닝: 자체 골드셋 30쌍("경복궁→창경궁·종묘·운현궁" 같은 상식 검증 쌍)을 만들어 nDCG@3로 그리드서치. 골드셋은 `data/goldset.jsonl`로 버전 관리.
- Fine-tuning **하지 않는다** — 데이터도 시간도 부족, 사전학습 임베딩으로 충분 (DECISION_LOG #6).

## 계층 3 — LLM (코스 생성·근거 문장화)

- 모델: Claude Haiku 4.5 (비용), 발표 데모 당일만 Sonnet 스왑 가능하게 모델명을 env로.
- **Agent 아님** — 단발 structured output 호출 2종 (코스 서사, 추천 이유). 도구호출·멀티턴 없음: 지연·실패 표면적 최소화.
- 환각 통제: LLM에 사실(운영시간·거리·혼잡)을 모두 컨텍스트로 주입하고 "제공된 정보만 사용" 제약. 출력은 JSON 스키마 검증 → 실패 시 1회 재시도 → 템플릿 폴백.
- Evaluation: 프롬프트 golden test 20케이스 (스키마 준수율 100%, 사실 위반 0건 수동 검수). PROMPTS.md에 프롬프트 버전 관리.

## AI 전체 성능 측정 (발표 슬라이드용 숫자 만들기)

1. 혼잡 예측: citydata 대비 등급 일치율 N% (서울 검증셋)
2. 대안 매칭: 골드셋 nDCG@3
3. 서비스 지표: 베타 사용자(지인 10명+) 추천 수용률, SUS 설문
- 세 숫자를 반드시 확보할 것 — "측정된 AI"는 학부생 팀 대부분이 못 하는 차별화 요소.
