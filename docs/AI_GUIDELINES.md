# AI_GUIDELINES — AI 작업자(LLM) 행동 규칙

> 이 프로젝트는 사람 1인 + AI 어시스턴트로 개발된다. 어떤 모델(Claude Sonnet/Opus, GPT 등)이 투입되어도 아래를 따른다.

1. **진입 절차**: PROJECT.md → 관련 문서 → TASKS.md에서 Task 선택. 문서에 없는 요구사항을 임의 발명하지 않는다.
2. **범위 규율**: MVP(PRD §6)에 없는 기능 제안은 구현하지 말고 PRD의 Future Roadmap에 한 줄 추가만 한다. "하는 김에" 금지.
3. **심사 정렬**: 모든 산출물은 4개 배점(구현성/기획/데이터/발전성) 중 무엇에 기여하는지 답할 수 있어야 한다.
4. **안정성 우선순위**: 에러 핸들링·폴백·캐시 > 새 기능 > 리팩토링 > 최적화.
5. **기록 의무**: 설계 이탈은 DECISION_LOG, 새 위험은 RISKS, 완료는 TASKS 갱신. 문서와 코드가 다르면 **문서를 고치는 커밋을 같이** 낸다.
6. **검증 의무**: "될 것이다"로 끝내지 않는다. 실행·호출·테스트 결과를 증거로 남긴다 (스크린샷/로그/테스트 통과).
7. **비밀 관리**: API 키·토큰을 코드·문서·커밋에 절대 포함하지 않는다.
8. **외부 데이터 규율**: DATA_GUIDELINES의 소스 대장에 없는 소스를 쓰려면 라이선스 확인 후 대장에 등록부터.
9. **한국어 출력**: 사용자 대면 텍스트·문서·커밋 메시지는 한국어 (코드 식별자는 영어).
10. **모델별 역할 제안**: 대량 보일러플레이트·CRUD = 경량 모델 / 아키텍처 변경·심사 전략·ML 설계 = 고성능 모델. 넘길 때 PROMPTS.md P9 사용.
