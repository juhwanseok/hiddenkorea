## 3. 전처리 규칙

- POI: `contentid` 기준 dedup, overview HTML 제거, 좌표 결측/0.0 제거, `firstimage` 없는 POI는 대안 추천 후보에서 후순위(δ 페널티).
- 시군구 코드 매핑: TourAPI areaCode/sigunguCode ↔ 행안부 법정동코드 ↔ 데이터랩 지역코드 — **매핑 테이블을 `data/region_map.csv`로 직접 구축** (소스마다 코드계가 다름, 최다 버그 예상 지점).
- 집중률(비율) → 지수(0~100) 정규화: POI별 30일 내 min-max가 아닌 **전 POI 공통 분위수 기반** (관광지 간 비교 가능해야 함).

## 5. 데이터 품질 관리

- 배치 잡마다 row count·결측률 assert, 20% 이상 급변 시 잡 실패 처리 + 이전 스냅샷 유지 (심사 중 데이터 오염 방지).
- 라이선스: TourAPI 공공누리 1·3유형 — 출처 표기 의무 (푸터에 "한국관광공사 TourAPI" 명시). 이미지 핫링크 대신 URL 사용(재배포 아님).
- 개인정보: 수집하는 개인정보 없음 (로그인 없음, 로그에 IP 저장 안 함) — 심사 질의 대비 한 줄 답변 준비.
- 증강: 하지 않음. 필요 시 프록시 타깃 확장(D2 커버 POI 수가 충분하면 불필요).
