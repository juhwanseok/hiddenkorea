# EVALUATION_GUIDELINES — 품질 측정

> 원칙: 발표에서 말할 모든 숫자는 재현 가능한 스크립트로 생성한다 (`ml/eval/`).

## 1. 혼잡도 예측
- **MAE** (지수 0~100), **Spearman ρ** (POI 간 순위), **등급 일치율** (5단계, 인접 허용)
- 프로토콜: 시간 분할 — 학습은 T-4주 이전, 평가는 최근 4주. 누수 금지(미래 피처 사용 금지)
- 외부 검증: 서울 citydata 실측 vs 우리 예측, 주간 자동 리포트 (`ml/eval/seoul_validation.py`) → 결과 JSON을 레포에 커밋 (추이 그래프 = 발표 슬라이드)

## 2. 대안지 매칭
- **nDCG@3** on goldset.jsonl (30쌍) — 가중치 튜닝의 목적함수
- 정성 검수: 지역 대표 관광지 20곳의 추천 결과를 표로 출력해 수동 O/X (`ml/eval/matching_review.md`)

## 3. LLM
- golden test 20케이스: JSON 스키마 100%, 사실 위반 0, 길이 준수 (PROMPTS.md 규약)

## 4. 서비스
- p95 API 응답 < 2s (캐시 히트 시 < 300ms), 크래시 프리 세션 ≥ 99%
- 베타(≥10명): 추천 수용률(대안 카드 클릭/노출), 코스 생성 완료율, SUS 점수

## 5. 심사 리허설 (W9)
- 제3자(지인)에게 URL만 주고 5분 자유 사용 → 막힌 지점 전부 이슈화. 2회 반복.
